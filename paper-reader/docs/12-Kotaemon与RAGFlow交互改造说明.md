# Kotaemon 与 RAGFlow 交互改造说明

## 一、改造目标

本轮改造聚焦论文阅读中的两个关键体验：其一，将原有卡片式论文知识结构升级为可缩放、可折叠、具有真实父子连线的思维导图；其二，将 MinerU 解析得到的文字、图片、表格和公式切片与 PDF 原文放在同一页面，用户点击切片即可同步定位到对应页并高亮原文区域。

## 二、参考项目及借鉴内容

### 1. Cinnamon/kotaemon

Kotaemon 的论文信息面板会将层级化内容转换为 Markdown 标题结构，再由 Markmap 渲染为可交互思维导图；其 PDF 阅读器还支持引用位置预览和原文高亮。本项目借鉴其“层级数据 + Markmap 可视化”的实现思路，但思维导图内容由本系统的论文理解 Agent 根据 MinerU 解析结果生成。

### 2. infiniflow/ragflow

RAGFlow 的文档解析结果页面采用切片列表和 PDF 原文并排布局，选中切片后由文档预览组件定位并高亮相应区域。本项目借鉴该交互方式，采用约 42% 的切片区与 58% 的原文区，避免用户在“结构化内容”和“阅读工作台”之间反复跳转。

## 三、系统实现

### 1. 多层思维导图

- 后端论文理解 Agent 输出中心主题、一级分支、二级子分支和三级知识点。
- 前端将结构化 JSON 转换为 Markdown 层级，再交给 Markmap 绘制节点、曲线和层级关系。
- 支持节点展开与折叠、拖动画布、滚轮缩放、按钮缩放、适应画布和全屏查看。
- 分支概述保留在导图下方，避免长文本撑大节点而破坏整体结构。
- 思维导图结果继续使用后端缓存；普通访问读取已有结果，“重新生成”才调用大模型刷新。

### 2. 切片与 PDF 同屏对照

- 左侧按图片、图表、表格、公式、正文筛选 MinerU 结构化切片。
- 每个切片展示页码、类型、结构化文字、LaTeX 或资源预览图。
- 点击切片只更新当前选择，不离开页面；右侧 PDF 自动跳转到对应页。
- 若 MinerU 返回 bbox 坐标，前端按照 PDF 显示比例换算坐标并覆盖高亮框。
- 当前切片使用高亮边框和“正在对照”状态，形成切片与原文的一一对应关系。

## 四、相关文件

- `backend/app/agents/paper_agent.py`：生成三级以上思维导图数据。
- `backend/app/models/schemas.py`：定义子分支数据模型并兼容旧缓存。
- `frontend/src/components/MindMapView.tsx`：Markmap 交互式思维导图。
- `frontend/src/components/StructuredContentView.tsx`：切片与原文同屏对照。
- `frontend/src/components/PaperReader.tsx`：按页码和 bbox 定位高亮。
- `frontend/src/styles.css`：双栏阅读和全屏导图样式。

## 五、验收要点

1. 选择已解析论文后，结构化内容页同时显示切片和 PDF，不发生页面跳转。
2. 点击不同页的切片，PDF 页码同步变化；存在 bbox 时显示对应高亮框。
3. 思维导图至少包含“中心主题—一级分支—二级分支—知识点”三级结构。
4. 思维导图支持展开/折叠、缩放、适应画布和全屏操作。
5. 刷新页面后能够读取已缓存的思维导图，重新生成后缓存随之更新。
