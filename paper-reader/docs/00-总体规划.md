# 多Agent论文智能阅读系统总体规划

## 1. 项目目标

在两周内完成一个可部署、可演示的论文阅读系统最小可行产品。核心闭环为：

> 上传论文 → MinerU解析 → 建立知识库 → 生成智能导读 → 论文聊天问答 → 原文页码定位 → 推荐并播放本地学习视频。

章节翻译、图表公式问答作为第二个Sprint的增强能力。系统至少体现协调Agent、论文理解Agent、翻译Agent和聊天问答Agent四个Agent的任务分工、调用过程及日志。

## 2. 参考项目与采用方式

### 2.1 opendatalab/MinerU

- 地址：https://github.com/opendatalab/MinerU
- 参考内容：PDF版面解析、阅读顺序恢复、图片/表格/公式提取，以及Markdown/JSON结构化输出。
- 项目采用方式：将MinerU封装为独立解析服务适配器，不复制其模型代码；开发阶段可使用PyPDF回退。

### 2.2 infiniflow/ragflow

- 地址：https://github.com/infiniflow/ragflow
- 参考内容：文档知识库、内容切片、检索增强问答、回答引用和Agent工作流。
- 项目采用方式：借鉴模块边界和交互，不直接基于完整RAGFlow二次开发。RAGFlow规模较大，不适合两周课程项目直接修改。

### 2.3 langchain-ai/langgraph

- 地址：https://github.com/langchain-ai/langgraph
- 参考内容：有状态Agent编排、任务路由、运行轨迹与可恢复工作流。
- 项目采用方式：第一阶段使用项目内轻量协调器稳定接口；第二阶段可将CoordinatorAgent内部替换为LangGraph状态图。

### 2.4 mozilla/pdf.js

- 地址：https://github.com/mozilla/pdf.js
- 参考内容：浏览器内PDF渲染、分页和原文页面定位。
- 项目采用方式：前端通过react-pdf封装PDF.js，在聊天引用被点击时跳转对应页。

## 3. 技术架构

| 层级 | 技术 | 主要职责 |
|---|---|---|
| 前端交互层 | React、TypeScript、Vite、PDF.js | 上传、阅读、导读、聊天、翻译、视频播放 |
| API层 | FastAPI | 统一接口、文件访问、参数校验、Agent日志 |
| Agent层 | Coordinator及三个专业Agent | 导读、翻译、聊天问答、视频推荐 |
| 文档处理层 | MinerU适配器、PyPDF回退 | PDF解析、页码和多模态内容元数据 |
| RAG层 | 可替换KnowledgeBase接口 | 内容切片、索引、检索和引用 |
| 资源层 | 本地文件、JSON目录，后续可换数据库 | PDF、视频、元数据和日志 |
| 部署层 | Docker Compose、Nginx | 前后端一键启动和静态资源代理 |

2. 论文库；
3. PDF阅读器；
4. 智能导读面板；
5. 聊天问答区；
6. 引用卡片与原文定位；
7. 视频推荐卡片和播放器；
8. 后续补充中英文对照页面及Agent日志页面。

## 5. 第一阶段验收

- 后端健康检查和OpenAPI文档可以访问；
- PDF可以上传并保存；
- 未部署MinerU时可以用PyPDF完成文字提取；
- 三个专业Agent均通过协调Agent调用；
- 聊天响应包含原文页码引用结构；
- 视频根据人工维护的关键词推荐并在网页播放；
- Agent调用日志可查询；
- 前后端可通过Docker Compose启动。
