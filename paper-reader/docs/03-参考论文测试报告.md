# 参考论文端到端测试报告

## 1. 测试目的

使用课程提供的8篇真实论文验证系统核心闭环，检查PDF是否能够上传和解析、知识库是否能够建立、论文理解Agent能否生成导读、聊天Agent能否返回带页码的引用，以及论文元数据和切片索引能否持久化。

## 2. 测试环境与方式

- 测试日期：2026年7月13日
- 解析方式：PyPDF开发回退
- 大模型：未配置，使用提取式导读和提取式回答
- 测试入口：FastAPI TestClient
- 数据目录：临时隔离目录，测试结束后自动清理
- 测试脚本：`backend/scripts/smoke_reference_papers.py`

每篇论文依次执行：

1. 调用上传接口并完成解析；
2. 验证页数大于0；
3. 调用论文理解Agent生成导读；
4. 提问“What is the main contribution of this paper?”；
5. 验证回答、引用、页码和Agent轨迹；
6. 验证论文记录与内容切片已写入持久化存储。

## 4. 已修正的问题

1. 增加论文元数据JSON持久化，后端重启后仍可恢复论文列表；
2. 增加按论文保存的切片索引，重启后可重新载入知识库；
3. 无大模型时不再只返回占位说明，而是输出论文摘要式导读；
4. 无大模型时聊天回答会展示检索到的论文依据和页码；
5. 用户提出通用“推荐视频”请求时，可以返回人工维护的本地视频资源；
6. 新增参考论文扫描脚本和端到端烟雾测试脚本。

## 5. 当前边界

- 本轮测试使用PyPDF验证文本PDF，尚未验证扫描版PDF和复杂OCR；
- 真实翻译、归纳式回答和高质量导读仍需配置大模型；
- 图片、表格和公式的结构化结果需要接入MinerU后继续测试；
- 当前知识库为轻量词项检索，后续需替换为嵌入模型和向量数据库；
- 测试耗时为本机开发回退模式结果，不代表MinerU或大模型服务耗时。

## 6. 下一步建议

下一阶段应先部署MinerU并用其中2篇图表较多的论文进行解析对比，确认文字、公式、图片和表格的统一数据结构；随后接入一个OpenAI兼容模型，完成真实导读、翻译和基于引用的问答质量测试。
