# 前端 NotebookLM 风格 UI/UX 深化说明

更新时间：2026-07-17
改造范围：前端全局视觉、阅读工作台三栏、顶部导航、右侧功能标签、智能导读、论文对话、结构化切片、中英对照、独立页面和移动端适配。
本轮不涉及 US-10 多 Agent 组合导读，也没有修改后端接口或数据结构。

## 一、文件级修改记录

| 文件 | 修改内容 | 实现效果 |
|---|---|---|
| `frontend/src/App.tsx` | 新增统一的 `NavIcon`、`BrandIcon`；顶部主导航和右侧四个功能标签接入线性图标；补充 `aria-current`、`aria-label` 和 `title` | 导航语义更清晰；桌面端展示“图标 + 文案”，窄屏可安全收缩为图标模式且不损失可访问名称 |
| `frontend/src/main.tsx` | 在旧样式和聊天专项样式之后引入 `styles/notebooklm-theme.css` | 使用最终主题层统一收口历史样式，避免直接大面积删除旧 CSS 带来的回归风险 |

## 二、视觉与交互改造

### 1. 整体视觉

- 页面底色调整为低饱和度浅蓝灰 `#f1f4fc`，主体内容使用纯白卡片。
- 三栏均使用 20px 圆角、细边框和柔和弥散阴影，形成清晰但克制的层级。
- 主色统一为低饱和度蓝 `#4169d8`，正文改为中性深灰，避免全界面“高纯度 AI 蓝”。
- 不下载外部字体，优先使用 `Inter / Noto Sans SC / 微软雅黑 / 苹方`，保证组员机器离线运行时显示稳定。
- 学术原文、证据摘录使用楷体/衬线回退栈；标题和操作区保持无衬线字体，避免全页面滥用楷体。

### 2. 顶部导航

- 品牌图标改为黑底白色线性文档图标，品牌名与产品副标题建立两级字号。
- 主导航选中态改为近黑色胶囊，未选中态为透明浅色按钮。
- 顶部导航标签增加轻微悬浮抖动：仅 1px 上移和约 0.45° 摆动，时长 280ms，不循环。
- 在 1040px 以下自动隐藏导航文字，仅保留图标；通过 `aria-label` 和 `title` 保留无障碍和鼠标提示。

### 3. 右侧四色功能标签

- 删除原来的四色圆点。
- 四个标签改为 2×2 功能卡布局，并以背景色区分：
  - 智能导读：淡紫；
  - 论文对话：淡绿；
  - 中英对照：淡橙；
  - 结构化切片：淡粉。
- 选中态增加边框、柔和阴影、字重和 1px 上浮；悬浮时保留各自颜色并增加轻微抖动。
- 标签宽度采用 `minmax(0, 1fr)`，长文案使用安全省略，避免挤压或截断布局。

### 4. 三栏与模块细节

- 左侧论文库：上传按钮改为 NotebookLM 式黑色主按钮；论文选中项使用浅蓝背景和左侧 3px 指示条；删除/重试操作仅在悬浮或键盘聚焦时显示，移动端始终可见。
- 中部阅读器：工具栏压缩为统一 36px 控件；分页、缩放、搜索框使用一致圆角和焦点环；PDF 舞台使用浅灰底与纸张柔影。
- 智能导读：“一句话概括”使用衬线字体和淡紫渐变卡片；各字段卡片增加悬浮上浮；关键词和引用标签统一为小字号胶囊。
- 论文对话：统一消息正文字号和行高；用户消息使用淡蓝底；定位证据按“编号、类型、页码、章节、原文摘录”分层；输入框使用粘底白卡和聚焦光环。
- 结构化切片：深色面板完全改为浅底白卡；筛选项、正文开关和 AI 解释区统一到同一设计体系。
- 中英对照：中文译文使用阅读字体，英文原文使用学术字体栈；段落悬浮和原文联动高亮更克制。
- Agent 日志、思维导图和拓展学习：页面头部、检索表单、结果卡和表格统一为白卡、18px 圆角和浅色悬浮反馈。

### 5. 动效与可访问性

- 普通控件动效统一为 140–190ms，卡片动效为 190ms，避免迟滞。
- 卡片悬浮最多上移 2px；主按钮最多上移 1px，避免夸张弹跳。
- 加载状态使用统一旋转指示和微光骨架屏。
- `prefers-reduced-motion: reduce` 下关闭动画和过渡。
- 全局 `:focus-visible` 使用 3px 半透明蓝色焦点环，键盘用户可明确定位。

## 三、响应式规则

- `1280px`：压缩左右栏默认宽度，保留完整三栏。
- `960px`：论文库和阅读器组成第一行，右侧助手移动到第二行并占满宽度。
- `720px`：工作台改为单栏；论文库、阅读器、助手依次纵向排列；功能标签保持 2×2；按钮触控高度不低于 40px。
- `480px`：隐藏品牌文字；右侧标签减小图标和字号；分页/缩放控件平均分配宽度。
- 空论文状态已纳入与阅读器相同的圆角、边框和最小高度规则，避免出现中间栏“方形白块”。

## 四、验证结果

- `npm run build`：通过。
- TypeScript `tsc --noEmit`：通过。
- Vite 生产构建：通过，818 个模块成功转换。
- 桌面端实测：1280×720 下三栏宽度正常，顶部和工作台均无横向溢出。
- 移动端实测：720×900 下单栏回流成功，页面 `scrollWidth` 与可视宽度一致，无横向滚动。
- 真实论文实测：PDF 阅读、智能导读、论文对话、四色功能切换正常。
- 浏览器控制台：0 条 error / warning。

构建仍会提示 `markmap-vendor` 大于 500 kB；该包已采用按需加载，只在进入思维导图页面时下载，不影响阅读工作台首屏。本轮未为消除提示而改变依赖或业务逻辑。

## 五、合并注意事项

1. `notebooklm-theme.css` 必须保持在 `base.css`、`styles.css`、`chat.css` 之后引入，否则旧样式会重新覆盖四色背景和最终字体层级。
2. 合并 `App.tsx` 时需同时保留 `NavIcon`、`BrandIcon` 及各按钮新增的 class、`aria-current`、`aria-label`。
3. 本轮没有新增 npm 依赖，不需要重新修改锁文件；正常执行 `npm install` 后即可运行。
4. 本轮只改变展示和交互，不改变现有 API、论文数据、聊天历史结构和 SSE 流程。
