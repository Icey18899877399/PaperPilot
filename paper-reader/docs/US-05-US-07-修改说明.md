# US-05 / US-07 功能修改说明

## US-05：论文智能问答与溯源

### 新增功能
- 多轮对话：同一论文的历史问答自动拼接为 LLM 上下文
- 对话管理：支持新建 / 切换 / 删除对话，切换论文自动重置
- 证据不足检测：LLM 输出 `INSUFFICIENT_EVIDENCE` 时替换为友好提示，前端显示 ⚠ 标识
- 对话持久化：JSON 文件存储，重启后历史可回看
- 输出格式规整：LLM 按 📌概括 → ●分点 → ①②编号 → 💡提示 的结构化模板输出
- 消息文本在页面中正确保留换行与缩进

## US-07：图片、图表和公式问答

### 新增功能
- 视觉查询自动分流：问"图X"/"表X"/"公式"时自动路由到视觉分析 Agent
- 图片/图表分析：识别类型、关键元素、核心信息，标注页码
- 表格分析：解释表头含义、对比关键数据，不编造数值
- 公式分析：输出 LaTeX 原文 + 逐项符号解释
- 证据不足保护：数据无关时 LLM 只输出 `INSUFFICIENT_EVIDENCE`，前端显示 ⚠
- 无 LLM 时降级为 MinerU 原文展示

### 新增文件（2个）
| 文件 | 说明 |
|------|------|
| `backend/app/services/visual_prompts.py` | 视觉关键词检测 + 4套专用 system prompt + 证据规则 |
| `backend/app/agents/visual_agent.py` | 视觉分析 Agent（分类→检索→构建上下文→LLM分析） |

### 修改文件（1个）
| 文件 | 改动 |
|------|------|
| `backend/app/api/routes/chat.py` | 视觉查询分流 + evidence_sufficient 逻辑修复 |

### 未改动的核心文件

## 新增 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/papers/{id}/conversations` | 列出论文下所有对话 |
| GET | `/api/papers/{id}/conversations/{cid}` | 获取单个对话详情 |

## 设计原则

本轮修改遵循**最小改动原则**：新功能通过新建文件实现，已有核心模块（Agent体系、路由层、调度器、LLM客户端）均未被修改。chat.py 作为唯一的集成点承担分流和编排职责。
