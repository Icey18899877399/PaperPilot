# 结构化内容浏览与思维导图说明

## 1. 结构化内容浏览

前端新增“结构化内容”入口，用于集中展示MinerU从当前论文中识别的图片、图表、表格、公式和正文。页面通过`GET /api/papers/{paper_id}/contents`获取数据，并提供以下能力：

- 按图片、图表、表格、公式和正文切换分类；
- 显示每种内容的识别数量；
- 展示图片、图表和表格截图；
- 以纯文本展示表格内容，以LaTeX展示公式；
- 显示内容所在页码；
- 点击“定位原文”后返回PDF阅读器，跳转到对应页并高亮`bbox`区域。

测试论文共识别图片2个、图表6个、表格13个、公式2个和正文170个。13个表格均带页内坐标，可用于原文定位。

## 2. 思维导图生成

后端新增`POST /api/papers/{paper_id}/mind-map`。请求由协调Agent路由给论文理解Agent，Agent从标题、章节标题、论文开头、中部和结尾抽取代表性内容，再调用DeepSeek生成JSON结构的思维导图。

返回结果包含：

- `center`：论文中心主题；
- `branches`：4—6个主要分支；
- `label`：分支名称；
- `summary`：分支概括；
- `children`：每个分支1—3个子要点；
- `agent_trace_id`：多Agent调用追踪编号。

提示词要求分支覆盖研究问题、核心方法、实验设计和主要结论。如果模型不可用或返回结构不完整，系统会自动生成4个基础分支，保证演示页面仍可使用。

## 3. 前端交互

前端新增“思维导图”入口。用户选择论文后点击“生成思维导图”，页面以中心主题和分支卡片形式展示结果。每个分支支持展开和收起子要点；切换到其他功能页再返回时，当前已生成结果会保留，避免重复调用模型。

## 4. 真实验收结果

使用`2025.coling-main.353.pdf`和`deepseek-v4-flash`进行真实测试：

| 验收项 | 结果 |
|---|---|
| 思维导图生成耗时 | 约6.9秒 |
| 中心主题 | 生成成功 |
| 主要分支 | 5个 |
| 子要点 | 15个 |
| Agent日志 | 记录`generate-mind-map`动作和Trace ID |
| 结构化表格 | 13个 |
| 结构化公式 | 2个 |
| 前端生产构建 | 通过 |

该成果可作为课程演示中的独立功能页，也可在PPT中用于说明“论文理解Agent如何把非结构化论文转换为知识结构”。
