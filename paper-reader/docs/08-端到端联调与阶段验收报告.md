# PaperPilot端到端联调与阶段验收报告

## 1. 验收范围

本阶段完成了测试论文从MinerU结构化解析结果入库，到DeepSeek导读、翻译、多模态问答、原文引用定位、视频推荐与网页内播放的端到端联调。测试论文为`2025.coling-main.353.pdf`，系统前端地址为`http://localhost:5173`，后端接口文档为`http://localhost:8000/docs`，MinerU服务地址为`http://127.0.0.1:8001`。

## 2. 结构化解析结果

测试论文共22页，形成196个可检索内容切片，包括正文170个、图片2个、表格13个、图表6个、列表3个、公式2个。结构化内容保留页码、类型、页内坐标和资源地址；表格保留HTML与纯文本，公式保留LaTeX，图片、图表和表格截图可以通过`/media/assets`访问。

为避免演示时重复等待同一论文的CPU解析，新增`scripts/register_mineru_result.py`。该脚本可将已完成的MinerU解析结果注册到正式论文库，同时保存原始PDF、论文记录和检索索引；正式上传接口仍可用于解析新论文。

## 3. DeepSeek与多Agent验收

运行时模型状态为`provider=deepseek`、`model=deepseek-v4-flash`、`configured=true`。真实接口测试结果如下：

| 功能 | 结果 | 单次实测耗时 |
|---|---|---:|
| 论文智能导读 | 生成概述、3个阅读重点和3个思考问题 | 约4.4秒 |
| 公式问答 | 优先命中2个公式切片，并返回页码引用 | 约6.1秒 |
| 表格问答 | 6条引用均为表格类型，覆盖第5、7、12、15、17页 | 约6.0秒 |
| 图表问答 | 6条引用均为图表类型，覆盖第5、7、8、12页 | 约3.9秒 |
| 学术翻译 | 返回真实模型译文，未进入开发回退模式 | 约2.0秒 |

Agent日志已记录协调Agent、论文理解Agent、翻译Agent和聊天问答Agent共4类角色，包含`route`、`read-paper`、`generate-guide`、`translate`和`answer`等动作以及Trace ID，满足“至少3个Agent”的可展示要求。

## 4. 前端演示成果

前端已完成生产构建，并提供以下三个可切换入口：

1. **阅读工作台**：上传或选择论文、PDF翻页、生成智能导读、段落翻译、基于论文聊天、点击引用跳转原文页码。
2. **Agent日志**：查看4类Agent的记录数量、执行动作、详情、时间和Trace ID。
3. **视频资源**：查看本地学习视频目录并直接在网页内播放。

项目内生成了`transformer-demo.mp4`作为12秒H.264本地播放样例，文件经后端与前端代理访问均返回HTTP 200。聊天问题中包含“Transformer”“注意力”或明确的视频推荐意图时，会返回该视频卡片并支持网页内播放。

## 5. 本阶段修复的问题

联调时发现MinerU配置脚本使用PowerShell默认UTF-8写入方式，会在`.env`首行加入BOM，导致`DEEPSEEK_API_KEY`变量名无法被Pydantic Settings识别。现已将`configure-mineru.ps1`改为无BOM UTF-8写入，并修复现有`.env`。修复后运行中模型状态与真实翻译、导读、问答均验证通过。
