# 多Agent论文智能阅读系统 — 部署文档

## 1. 项目概述

PaperPilot 是一个多Agent论文智能阅读系统，支持 PDF 上传与结构化解析、智能导读生成、思维导图、中英双语对照翻译、多轮对话问答（含图片/图表/表格/公式的视觉分析）、原文引用定位与证据溯源、扩展学习资源搜索，以及本地学习视频推荐与播放。

- 前端：React 19 + TypeScript + Vite + PDF.js + markmap
- 后端：FastAPI + Python 3.11+
- 文档解析：MinerU（推荐）/ PyPDF（回退）
- LLM：DeepSeek API（兼容 OpenAI 接口）
- 检索：轻量内存知识库，支持关键词与章节加权评分
- 视频：本地 MP4 元数据匹配

## 3. 快速启动（开发模式）

### 3.1 克隆项目

```powershell
git clone git@github.com:Icey18899877399/PaperPilot.git
cd PaperPilot\paper-reader
```

### 3.2 配置环境变量

```powershell
copy .env.example .env
# 编辑 .env 填入 DeepSeek API Key
```

### 3.3 启动后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
uvicorn app.main:app --reload
```

后端启动在 `http://localhost:8000`，API 文档位于 `http://localhost:8000/docs`。

### 3.4 启动前端

```powershell
cd frontend
npm install
npm run dev
```

前端启动在 `http://localhost:5173`。

### 3.5 验证

浏览器打开 `http://localhost:5173`，上传一篇 PDF 论文，等待解析完成后测试导读、问答、翻译功能。

## 4. Docker Compose 部署

项目根目录下有 `docker-compose.yml`，包含前后端服务：

```powershell
docker compose up -d
```

| 服务 | 端口 | 说明 |
|------|------|------|
| backend | 8000 | FastAPI 后端 |
| frontend | 5173:80 | React 前端（Nginx 静态服务） |

> Docker 模式下需要在 `.env` 中配置 `DEEPSEEK_API_KEY`，且 MinerU 需单独部署。

## 5. MinerU 结构化解析部署（可选）

MinerU 是独立的 Docker 服务，用于将 PDF 解析为结构化数据（正文、图片、表格、公式、图表）。不部署时系统自动降级为 PyPDF 纯文本提取。

### 5.1 首次安装

```powershell
cd <mineru-docker目录>
powershell -ExecutionPolicy Bypass -File setup.ps1
```

该脚本会构建 MinerU 镜像并下载 pipeline 模型。

### 5.2 日常启动

```powershell
cd <mineru-docker目录>
docker compose up -d
```

MinerU 服务启动在 `http://127.0.0.1:8001`。

### 5.3 配置项目连接 MinerU

在 `.env` 中设置：

```
MINERU_API_URL=http://127.0.0.1:8001
```

验证连接：

```powershell
cd backend
python scripts\smoke_mineru_reference.py "D:\软件工程_小学期\reference\your-paper.pdf"
```

## 6. LLM 配置

### 6.1 DeepSeek（推荐）

运行安全配置脚本，密钥输入不可见：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\configure-deepseek.ps1
```

或手动编辑 `.env`：

```
DEEPSEEK_API_KEY=sk-your-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_THINKING=false
```

验证连接：

```powershell
cd backend
python scripts\test_deepseek_connection.py
```

### 6.2 其他 OpenAI 兼容模型

```
LLM_BASE_URL=https://api.openai.com/v1
LLM_API_KEY=sk-your-key
LLM_MODEL=gpt-4o-mini
```

> DeepSeek 配置优先：当 `DEEPSEEK_API_KEY` 有效时自动使用 DeepSeek。

## 10. 常见问题

**Q: 后端启动报 "cannot import name"**
A: 检查 Python 版本是否为 3.11+，依赖是否完整安装（`pip install -e ".[dev]"`）。

**Q: 前端启动后页面空白或 API 请求失败**
A: 确认后端已启动在 8000 端口。前端 Vite 配置了代理 `/api` → `localhost:8000`，若后端端口不同需修改 `frontend/vite.config.ts`。

**Q: GitHub 连接超时**
A: HTTPS 端口（443）可能被防火墙阻断，改用 SSH 协议：
```powershell
git remote set-url origin git@github.com:Icey18899877399/PaperPilot.git
```

**Q: MinerU 解析失败或超时**
A: 确认 Docker 已启动且 MinerU 容器运行中：
```powershell
docker compose ps
docker compose logs -f
```
若 MinerU 不可用，系统会自动降级为 PyPDF 纯文本提取。

**Q: DeepSeek API 返回错误**
A: 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否正确，运行连接测试：
```powershell
cd backend
python scripts\test_deepseek_connection.py
```

**Q: 论文问答返回"证据不足"**
A: 这是系统的保护机制，表示 LLM 判断检索到的论文内容不足以支持可靠回答。尝试换一种更具体的提问方式。
