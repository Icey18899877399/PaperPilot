# PaperPilot 部署文档

## 1. 部署范围

PaperPilot 由 React 前端、FastAPI 后端和可选的 MinerU 解析服务组成。后端负责论文上传、解析、检索、导读、翻译、问答和学习资源检索，运行数据保存在 `backend/data`。前端开发服务器默认监听 5173 端口，后端默认监听 8000 端口，MinerU 默认监听 8001 端口。

项目支持两种运行方式：

- 本地开发：分别运行 Python 后端和 Vite 前端，适合开发与调试。
- Docker Compose：构建前后端镜像后统一运行，适合验收和固定环境部署。

MinerU 不是启动主程序的必要条件。未配置 MinerU 时，后端使用 PyPDF 提取纯文本；需要图片、表格、公式和版面坐标时再部署 MinerU。

## 2. 环境准备

### 2.1 基础软件

| 软件 | 建议版本 | 用途 |
| --- | --- | --- |
| Git | 2.40 及以上 | 获取代码 |
| Python | 3.11 或 3.12 | 运行后端和测试 |
| Node.js | 22 LTS | 构建和运行前端 |
| npm | 随 Node.js 安装 | 安装前端依赖 |
| Docker Desktop | 24 及以上，可选 | 容器部署和 MinerU |
| PowerShell | 5.1 或 7 | 执行仓库脚本 |

首次安装后执行以下命令确认工具可用：

```powershell
git --version
python --version
node --version
npm --version
docker version       # 仅 Docker 部署需要
docker compose version
```

MinerU CPU 镜像和模型占用空间较大，建议预留 20 GB 以上磁盘空间和至少 8 GB 内存。解析图表较多的论文时，16 GB 内存更稳妥。

### 2.2 获取代码

```powershell
git clone git@github.com:Icey18899877399/PaperPilot.git
cd PaperPilot\paper-reader
```

没有配置 GitHub SSH 密钥时可使用 HTTPS：

```powershell
git clone https://github.com/Icey18899877399/PaperPilot.git
cd PaperPilot\paper-reader
```

后续更新代码前先确认本地没有未提交修改：

```powershell
git status
git pull --ff-only origin main
```

## 3. 首次初始化

### 3.1 创建环境配置

在 `paper-reader` 目录执行：

```powershell
Copy-Item .env.example .env
```

`.env` 已被 Git 忽略，不应提交真实密钥。不开通模型和 MinerU 也能启动项目，但导读、翻译和问答会使用提取式结果或显示待配置提示。

最小开发配置如下：

```dotenv
APP_ENV=development
BACKEND_PORT=8000
FRONTEND_ORIGIN=http://localhost:5173
MAX_PDF_SIZE_MB=50

DEEPSEEK_API_KEY=
MINERU_API_URL=
```

需要 DeepSeek 时，建议使用仓库脚本写入密钥，输入内容不会显示在终端：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\configure-deepseek.ps1
```

脚本默认写入 `DEEPSEEK_BASE_URL=https://api.deepseek.com`、`DEEPSEEK_MODEL=deepseek-v4-flash` 和 `DEEPSEEK_THINKING=false`。模型名称应以账户实际可用的 DeepSeek API 模型为准；若接口提示模型不存在，应修改 `.env`，不要修改业务代码。

也可配置其他 OpenAI Chat Completions 兼容服务：

```dotenv
DEEPSEEK_API_KEY=
LLM_BASE_URL=https://your-provider.example/v1
LLM_API_KEY=your-key
LLM_MODEL=your-model
```

只要 `DEEPSEEK_API_KEY` 非空，DeepSeek 配置就优先于通用的 `LLM_*` 配置。

### 3.2 初始化后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
cd ..
```

如果 PowerShell 禁止激活脚本，可仅对当前用户放开本地脚本执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

也可不激活环境，后续直接使用 `backend\.venv\Scripts\python.exe`。

### 3.3 初始化前端

```powershell
cd frontend
npm ci
cd ..
```

仓库包含 `package-lock.json`，首次安装和持续集成应优先使用 `npm ci`，以保证依赖版本一致。

### 3.4 数据目录

后端首次导入配置时会自动创建上传、索引、导读、思维导图、翻译和解释缓存目录，无需手动建表或执行数据库迁移。本项目当前使用 JSON 文件持久化，没有 MySQL、PostgreSQL 或 Redis 初始化步骤。

主要数据位置：

| 路径 | 内容 |
| --- | --- |
| `backend/data/papers.json` | 论文元数据 |
| `backend/data/uploads/` | 上传的 PDF |
| `backend/data/assets/` | MinerU 图片及结构化产物 |
| `backend/data/indexes/` | 切片和可选向量缓存 |
| `backend/data/guides/` | 导读缓存 |
| `backend/data/mindmaps/` | 思维导图缓存 |
| `backend/data/translations/` | 双语页面缓存 |
| `backend/data/explanations/` | 切片解释缓存 |
| `backend/data/videos/catalog.json` | 视频资源目录 |

部署迁移或备份时，应整体保留 `backend/data`。不要在保存数据后随意手工修改 JSON 文件。

## 4. 本地开发启动

### 4.1 分别启动前后端

打开第一个 PowerShell 窗口：

```powershell
cd PaperPilot\paper-reader\backend
.\.venv\Scripts\Activate.ps1
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

打开第二个 PowerShell 窗口：

```powershell
cd PaperPilot\paper-reader\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

访问地址：

- 前端：`http://localhost:5173`
- 后端健康检查：`http://localhost:8000/api/health`
- OpenAPI 文档：`http://localhost:8000/docs`

Vite 已将 `/api` 和 `/media` 代理到 `localhost:8000`，本地开发无需额外配置前端 API 地址。

### 4.2 使用一键开发脚本

完成后端和前端依赖初始化后，也可在 `paper-reader` 目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\dev.ps1
```

该脚本在后台启动两个进程，并把日志写入：

- `dev-backend.out.log`、`dev-backend.err.log`
- `dev-frontend.out.log`、`dev-frontend.err.log`

脚本不会安装依赖，也不会自动停止之前启动的同端口进程。端口占用时先结束旧进程，再重新运行。

## 5. MinerU 结构化解析

### 5.1 选择部署方式

仓库提供两套 MinerU 方案，二选一即可：

- `mineru-docker/`：固定 MinerU 3.4.0 CPU 环境，适合可复现部署。
- `paper-reader/scripts/install-mineru.ps1`：安装到本地 Python 虚拟环境，适合不使用 Docker 的 Windows 主机。

不要同时在 8001 端口启动两套服务。

### 5.2 Docker 方式

从仓库根目录执行：

```powershell
cd mineru-docker
powershell -ExecutionPolicy Bypass -File setup.ps1
```

首次运行会构建镜像并下载 pipeline 模型，耗时取决于网络和磁盘性能。后续启动只需：

```powershell
cd mineru-docker
docker compose up -d
docker compose ps
```

验证服务：

```powershell
Invoke-RestMethod http://127.0.0.1:8001/health
```

本地运行 FastAPI 时，在 `paper-reader/.env` 中设置：

```dotenv
MINERU_API_URL=http://127.0.0.1:8001
MINERU_BACKEND=pipeline
MINERU_LANGUAGE=ch
MINERU_TIMEOUT_SECONDS=3600
```

也可执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\configure-mineru.ps1
```

### 5.3 本地 Python 方式

```powershell
cd paper-reader
powershell -ExecutionPolicy Bypass -File scripts\install-mineru.ps1
powershell -ExecutionPolicy Bypass -File scripts\start-mineru.ps1
```

安装脚本会创建独立的 `.venv-mineru`，不会污染后端 `.venv`。服务启动后另开终端配置项目并检查健康状态：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\configure-mineru.ps1
powershell -ExecutionPolicy Bypass -File scripts\test-mineru-health.ps1
```

### 5.4 Docker 后端访问 MinerU

若 PaperPilot 后端运行在 Docker 容器内，而 MinerU 通过宿主机的 8001 端口提供服务，`.env` 必须使用：

```dotenv
MINERU_API_URL=http://host.docker.internal:8001
```

容器中的 `127.0.0.1` 指向容器自身，不能访问宿主机 MinerU。修改 `.env` 后需要重新创建后端容器。

## 6. Docker Compose 部署前后端

先确保已创建 `.env`，然后在 `paper-reader` 目录执行：

```powershell
docker compose build
docker compose up -d
docker compose ps
```

服务端口：

| 服务 | 宿主机地址 | 说明 |
| --- | --- | --- |
| frontend | `http://localhost:5173` | Nginx 托管前端并代理 API |
| backend | `http://localhost:8000` | FastAPI，可直接查看接口文档 |
| MinerU | `http://localhost:8001` | 单独部署时存在 |

查看日志：

```powershell
docker compose logs --tail 100 backend
docker compose logs --tail 100 frontend
```

停止服务但保留数据：

```powershell
docker compose down
```

`backend/data` 通过卷挂载到容器，重新构建镜像不会删除论文数据。部署到其他主机时应限制 8000 端口的公网访问，并在外层反向代理配置 HTTPS、上传大小限制和访问控制。

## 7. 可选能力配置

### 7.1 远程向量服务

默认检索使用内置 TF-IDF，不需要额外服务。需要跨语言语义检索时，可配置 OpenAI Embeddings 兼容接口：

```dotenv
EMBEDDING_API_URL=https://your-provider.example/v1
EMBEDDING_API_KEY=your-key
EMBEDDING_MODEL=your-embedding-model
```

接口失败时会自动回退到本地 TF-IDF。向量结果缓存在 `backend/data/indexes/*.vectors.json`。

### 7.2 视觉模型

MinerU 可提供图片资源和图题；如需把真实图片发送给多模态模型解释，可配置 OpenAI Chat Completions 兼容视觉接口：

```dotenv
VISION_API_URL=https://your-provider.example/v1
VISION_API_KEY=your-key
VISION_MODEL=your-vision-model
```

三项均配置时才启用。视觉接口不可用时，系统回退到图题和相邻正文的文本解释。

### 7.3 扩展学习资源

OpenAlex 和 Crossref 的基础检索不强制要求密钥。可选配置如下：

```dotenv
OPENALEX_API_KEY=
CROSSREF_MAILTO=maintainer@example.com
LEARNING_SEARCH_TIMEOUT_SECONDS=20
```

## 8. 启动后验证

### 8.1 基础检查

```powershell
Invoke-RestMethod http://localhost:8000/api/health
Invoke-RestMethod http://localhost:8000/api/models/status
```

健康接口应返回 `status: ok`。模型状态接口不会返回 API 密钥。

### 8.2 自动测试

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
python -m pytest tests -v
```

前端构建检查：

```powershell
cd frontend
npm run build
```

### 8.3 首次业务验收

1. 打开 `http://localhost:5173`，上传一个小于 `MAX_PDF_SIZE_MB` 的有效 PDF。
2. 等待论文状态变为“解析完成”，检查 PDF 页面和结构化内容。
3. 在未配置 LLM 时确认提取式降级可用；配置 LLM 后生成导读并测试带页码引用的问答。
4. 生成一页双语对照内容，确认刷新后仍能读取缓存。
5. 配置 MinerU 后上传包含图表或公式的新论文，确认结构化块和资源图片可访问。
6. 删除测试论文，确认列表、上传文件及派生缓存同步清理。

## 9. 常见问题

### 后端提示模块不存在

确认当前目录是 `paper-reader/backend`，并使用该目录下的虚拟环境执行 `python -m pip install -e ".[dev]"`。

### 前端能打开但接口请求失败

本地开发检查 8000 端口和 `dev-backend.err.log`。前后端分开部署时设置 `VITE_API_BASE_URL` 指向后端公开地址，并将后端 `FRONTEND_ORIGIN` 改为前端的完整来源地址后重新构建前端。

### Docker Compose 报找不到 `.env`

在 `paper-reader` 目录执行 `Copy-Item .env.example .env`。Compose 文件明确引用该文件，即使暂时不配置模型也必须创建。

### 上传后一直处于解析中或解析失败

查看后端日志，并检查 MinerU 健康接口。首次 CPU 解析可能需要数分钟。若不要求结构化解析，可清空 `MINERU_API_URL`，重启后端，再对失败记录执行重新解析。

### 模型连接失败

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
python scripts\test_deepseek_connection.py
```

重点检查 API 地址、模型名称、密钥权限、账户余额和网络。不要把密钥粘贴到日志、截图或版本库。

### 修改 `.env` 后没有生效

配置在进程启动时读取。重启本地后端；Docker 部署则执行：

```powershell
docker compose up -d --force-recreate backend
```

### 端口被占用

```powershell
Get-NetTCPConnection -LocalPort 5173,8000,8001 -ErrorAction SilentlyContinue
```

结束确认无用的旧进程，或修改启动端口及对应代理配置。当前 `BACKEND_PORT` 只是配置项，仓库启动命令仍显式使用 8000 端口；改端口时必须同步修改 Uvicorn 命令、Vite/Nginx 代理和访问地址。
