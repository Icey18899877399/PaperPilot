# 后台解析进度与引用高亮说明

## 1. 改进目标

MinerU在CPU环境下解析22页论文约需6分钟。原同步上传会让浏览器长时间等待，用户无法判断系统是否仍在工作。本阶段增加后台解析模式、状态轮询和估算进度，同时利用MinerU返回的页码与`bbox`坐标实现聊天引用区域高亮。

## 2. 后台解析流程

前端上传调用`POST /api/papers?background=true`。后端保存PDF后立即返回论文记录，初始状态为`parsing`、进度为5%、阶段为“PDF已上传，等待解析”，随后由FastAPI后台任务完成以下步骤：

1. 调用MinerU进行文字、图片、表格、图表和公式解析；
2. 每15秒更新一次估算进度，最高到75%；
3. 解析完成后将进度更新到85%，构建论文检索索引；
4. 索引与结构化切片保存成功后更新为`ready`、100%和“解析完成”；
5. 出现异常时更新为`failed`、100%和“解析失败”，同时保存错误信息。

为保持已有自动化脚本兼容，未携带`background=true`时仍使用同步模式。历史论文记录会在加载时自动迁移：`ready`记录显示100%和“解析完成”，不会出现“已解析但0%”。

前端每2秒调用`GET /api/papers/{paper_id}`查询状态，并在论文列表中显示阶段文字和进度条。页面刷新后也会继续监控仍处于`parsing`状态的论文。

## 3. 引用区域高亮

MinerU切片保留`page`和`bbox=[x0,y0,x1,y1]`。聊天Agent把这两个字段放入引用信息。用户点击引用卡片后：

1. PDF阅读器切换到引用页码；
2. 根据PDF原始页面宽度和当前720像素渲染宽度计算缩放比例；
3. 将`bbox`转换为网页坐标；
4. 使用半透明黄色区域、橙色边框和短暂动画标出对应内容。

测试论文第5页表格坐标示例为`[137,82,465,179]`，PDF原始页面尺寸约为`595×842`，与MinerU坐标处于同一页面坐标系，可直接按渲染宽度缩放。

## 4. 验收结果

- 后台上传独立回归：初始状态`parsing`、5%，最终状态`ready`、100%；
- 历史论文兼容：当前2篇已解析论文均返回`progress=100`；
- 前端TypeScript检查和Vite生产构建通过；
- PaperPilot后端、DeepSeek、MinerU和前端服务健康检查通过；
- 图片、表格、图表和公式切片均保留页码与坐标，可用于跳页和高亮。

当前后台任务适合课程MVP和单机演示。如果后续需要多人同时上传或服务重启后自动恢复任务，应将后台任务升级为Redis加Celery/RQ等持久化任务队列。
