# US-01 上传并解析论文闭环验收报告

## 一、完成范围

本轮完成 US-01 剩余功能：PDF 文件大小校验、重复论文检测、失败论文重新解析以及解析完成后自动生成智能导读。原有后台解析、进度轮询、状态持久化和 MinerU 结构化解析能力保持不变。

## 二、实现内容

### 1. 文件校验

- 前端选择或拖拽文件时校验扩展名和文件大小。
- 后端再次校验 PDF 文件头，防止仅修改扩展名绕过检查。
- 默认最大文件大小为 50MB，可通过 `MAX_PDF_SIZE_MB` 修改。
- 超过限制返回 HTTP 413，无效 PDF 返回 HTTP 400，失败文件不会写入论文库。

### 2. 重复论文检测

- 上传时计算完整文件内容的 SHA-256，而不是只比较文件名。
- 同一论文即使重命名后再次上传，也会返回 HTTP 409。
- 错误信息返回已有论文名称和论文 ID；已有论文失败时提示使用“重新解析”。
- 旧版论文记录首次参与检测时会自动补充文件哈希和文件大小。

### 3. 失败后重新解析

- 论文列表中的失败记录显示“重试”按钮。
- 重试复用原始 PDF，不要求用户重新选择文件。
- 重试前清理旧切片、索引、解析资源、导读和思维导图，避免重复数据。
- 只有 `failed` 状态允许重试；解析中或已完成论文调用重试接口返回 HTTP 409。
- 重试过程继续展示阶段和进度，单篇失败不会影响其他论文。

### 4. 自动生成导读

- MinerU 解析及索引完成后，状态进入“自动生成论文导读”。
- 论文理解 Agent 自动生成导读并保存到 `backend/data/guides`。
- 前端在论文状态变为 `ready` 后自动读取已保存导读。
- `GET /api/papers/{paper_id}/guide` 读取缓存；`POST` 支持生成或刷新导读。
- 导读生成失败不会破坏已完成的解析结果，用户仍可在界面手动重试导读。

## 四、相关接口

- `POST /api/papers?background=true`：上传、校验、去重并启动后台解析。
- `GET /api/papers/{paper_id}`：查询阶段、进度和错误信息。
- `POST /api/papers/{paper_id}/retry?background=true`：重新解析失败论文。
- `GET /api/papers/{paper_id}/guide`：读取持久化导读。
- `POST /api/papers/{paper_id}/guide?refresh=true`：重新生成导读。
